# FigDataX — 高精度科学图表数据提取

**FigDataX** = **Fig**ure **Data** e**X**traction

从科学论文图表中精确提取数值数据，以多点轴校准半自动方法为核心，精度可达 ±0.5%。

> 本文件仅供人工阅读参考，不影响 skill 运行。技术细节见 `SKILL.md`。

---

## 安装与路径

### 安装为 Claude Code Skill

```bash
# 复制到 Claude Code skills 目录
cp -r figdatax ~/.claude/skills/
```

### 使用方式

**在 Claude Code 中使用（推荐）：**

直接告诉 Claude Code 提取图片数据即可：

```
> 提取 /path/to/figure.png 图片数据
> Extract data from /path/to/papers/fig3.png
> 从 ./results/figure2a.png 中读取数据
```

**输入**：提供图片文件的绝对或相对路径（PNG、JPG 等）

**输出**：所有生成文件保存在**输入图片所在的目录**（不是 skill 目录）

| 输出文件 | 说明 |
|----------|------|
| `{图片名}_extracted.csv` | 提取的数据表 |
| `{图片名}_validation.png` | 原图 vs 重建图对比验证 |
| `{图片名}_grid.png` | 坐标网格叠加图（中间文件） |

示例：
```
输入：~/papers/fig3.png
输出：~/papers/fig3_extracted.csv
      ~/papers/fig3_validation.png
      ~/papers/fig3_grid.png
```

**批量提取**：指向包含多张图的文件夹：
```
> 提取 /path/to/figures/ 文件夹中所有图片的数据
```

## 提取方法

| 方法 | 名称 | 适用场景 | 典型精度 |
|------|------|----------|----------|
| **M1** | **校准半自动** | **所有图表 — 默认首选** | **±0.5-2%** |
| M2 | 全自动颜色分割 | 高对比度、颜色分明的图表 | ±0.5-1% |
| M3 | Hough + 曲线追踪 | 折线图、连续曲线 | ±0.5-1% |

**策略**：始终使用 M1（校准半自动）。这是最精确的方法，因为它依赖用户提供的精确坐标轴参考点，而非 AI 猜测。M2/M3 仅在高对比度干净图表的批量自动化处理中作为补充。

---

## M1 核心流程（7 步）

### 步骤 1：加载并查看图像

用 Read 工具查看图像，识别：
- **图表类型**：柱状图、分组/堆叠柱状图、折线图、散点图、箱线图、热力图、饼图、极坐标图等
- **坐标轴**：标签、单位、刻度类型（线性 / 对数 / 双对数 / 倒数 / 日期）
- **刻度值**：两条坐标轴上每个刻度的精确数值
- **数据系列**：数量、图例标签、视觉编码（颜色、标记形状、线型）
- **标记点**：形状（圆形、菱形、方形、三角形）、大小、填充 — **标记点的几何中心才是真正的数据点**，而非其边缘
- **网格线**：主要和次要网格线（颜色检测前可能需要先去除）

### 步骤 2：检测或指定绘图区域

先尝试自动检测 `auto_detect_plot_area()`。若失败则手动确定像素坐标。

### 步骤 3：多点轴校准（精度关键）

每条轴使用 **3 个以上**刻度参考点，通过最小二乘拟合校正误差。这是精度最关键的一步。

向用户询问坐标轴刻度值，从图像中确定每个刻度的像素坐标，调用 `calibrate_axes_multipoint()`，检查 RMSE。

### 步骤 4：去除网格线（如需）

网格线会干扰颜色检测。使用 `remove_grid()` 去除，支持 Hough 线检测、颜色匹配和自适应方法。

### 步骤 5：提取数据点

**核心原则：标记点的几何中心才是数据点。**

大标记（直径 10-20 像素）如果读取边缘而非中心，可引入 5-10% 的误差。

**推荐方法 — 坐标网格叠加图 + 手动读取：**
1. 在原图上叠加 10px 间距的细网格 + 像素坐标标注（使用 `generate_grid_overlay()`）
2. 查看叠加图，读取每个标记**中心**的像素坐标
3. 用校准函数将像素坐标转换为数据值

此方法在以下场景尤为必要：
- 标记点较大
- 多条曲线共用同一颜色（如实线 vs 虚线黑色曲线，自动颜色分离失败）
- 图例区域与绘图区域重叠（产生误检）

**自动提取**（标记颜色可区分时）：使用 `extract_by_color_adaptive()` 自动检测色块质心。

**图例污染**：如果图例框与绘图区重叠，其文字/符号会产生误检。提取前需排除图例边界框区域。

### 步骤 6：像素坐标转数据值

将检测到的质心坐标通过校准函数转换为实际数据值，生成 DataFrame 并导出 CSV。

### 步骤 7：生成验证图

**必须生成验证叠加图**（原图 vs 提取数据重建图并排对比），这是发现错误最可靠的方式。

## M2 全自动颜色分割

适用于颜色分明的高对比度干净图表。精度低于 M1（使用简单边界框校准而非多点校准）。

- `detect_data_colors()` — K-means 自动检测绘图区主要颜色
- `auto_extract_bars()` — 自动提取柱状图（支持分组/堆叠）
- `auto_extract_scatter()` — 自动提取散点

## M3 Hough + 曲线追踪

适用于平滑曲线的折线图，使用霍夫线变换 + 轮廓曲线追踪 + 三次样条插值。

- `detect_axes_hough()` — 自动检测坐标轴
- `trace_curve()` — 按颜色追踪曲线（亚像素插值）
- `interpolate_curve()` — 稀疏点样条插值

## 特殊情况处理

| 场景 | 处理方式 |
|------|----------|
| 对数坐标轴 | `x_log=True, y_log=True` |
| 倒数坐标轴（如波数） | `x_transform="reciprocal"` |
| 误差线 / 置信区间 | `extract_error_bars()` |
| 分组 / 堆叠柱状图 | `group_detection=True, stacked=True` |
| 多面板图 (a, b, c, d) | `split_panels()` 分割后逐个用 M1 处理 |
| 极坐标图 | `extract_polar()` |
| 双 Y 轴图表 | 左右 Y 轴分别独立校准和提取 |

## 效率指南 — 精度与成本的平衡

**核心原则：一次脚本完成提取。** 读图→识别类型→单脚本完成校准+检测+提取+验证，避免多轮迭代探索浪费 tokens。

### 何时简化

| 情况 | 快捷方式 | 可跳过 |
|------|----------|--------|
| **分类X轴**（时间标签、组名） | 直接用类别索引，仅校准Y轴 | X轴校准 |
| **无网格线** | 跳过网格去除 | 步骤4 |
| **颜色可区分** | 自动颜色检测 | 手动网格读取 |
| **全黑同色标记** | 形态学腐蚀分离（见下文） | 颜色检测 |

### 网格密度

使用 **10px细网格 + 50px中网格 + 200px主网格**（3级）。10px 间距可达 ±5px 读取精度（约 ±0.01 数据单位），每个标记（10-20px）有1-2条细格线穿过，便于精确定位中心。50px 中网格带坐标标注方便快速查找。**不要**用 2-3px 超细网格（会遮盖底图）。

---

## 同色多系列检测（形态学方法）

当所有曲线共用同一颜色（如全黑，仅靠线型/标记形状区分），颜色检测失效。用**形态学腐蚀/膨胀**分离标记和连线。

### 流程
1. 二值化阈值 `< 100` → 暗像素 = 标记 + 线条 + 文字
2. 裁剪至绘图区，排除图例
3. 腐蚀（4×4核，1次迭代）→ 去除细线（~2-3px），保留粗标记（~8-15px）
4. 膨胀恢复标记大小
5. `findContours` → 按面积(60-300)和长宽比(0.6-1.5)过滤

### 曲线交叉处理

按 X 位置分组后，**不能假设固定的上下顺序**分配系列。需追踪曲线趋势，在交叉点后交换分配顺序。例如：12:00 前 Autumn > Summer，15:00 后 Summer > Autumn。

### 标记重叠

两系列值接近时（如都≈0.55），标记重叠成一个 blob，检测数少于预期。需手动补充缺失值或用插值。

### 空心标记

空心标记（仅有轮廓，中心白色）的几何中心是白色像素。`cv2.moments()` 可正确计算轮廓环的质心 = 几何中心。

## 精度最佳实践

1. **分辨率**：索要最高分辨率图像，PDF 导出用 300+ DPI PNG
2. **多点校准**：每轴 3+ 刻度，而非仅最小/最大值，校正透视畸变（精度最关键的一步）
3. **校准 RMSE**：若偏高（>轴范围 1%），检查刻度是否误识别
4. **标记中心 = 数据点**：大标记读边缘而非中心可引入 5-10% 误差，务必瞄准几何中心
5. **先去网格**：自动颜色检测前务必去除网格线
6. **亚像素精修**：始终启用 `subpixel=True`
7. **双 Y 轴**：各自独立校准
8. **图例污染**：排除图例区域以免误检
9. **同色曲线**：用形态学腐蚀分离标记（见上文），而非颜色检测
10. **曲线交叉**：在分配标记到系列时追踪交叉点，交叉后交换分配顺序
11. **重叠标记**：检测数少于预期时，手动补充或插值
12. **报告不确定度**：每次输出注明方法和校准 RMSE
13. **验证叠加图**：必须生成，最可靠的错误检查方式
